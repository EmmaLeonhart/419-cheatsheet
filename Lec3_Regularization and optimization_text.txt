Gradient Descent
▪Recall: the “gradient” of a function 𝑓: 𝑅𝑛 →𝑅 at a point 𝑝 is a 
vector in 𝑅𝑛 containing the partial derivatives of 𝑓 with respect 
to each input variable.
▪In other words, the gradient is an 𝑛-dimensional vector that tells 
us the direction and rate of fastest increase.
▪Key idea: pick a random starting point in the parameter space of 
𝑊, then repeatedly take steps in the direction of the negative of 
the gradient. 
▪Why negative of the gradient?
• The gradient tells you which way is uphill but you want to get to the 
valley (minimum), so you walk in the opposite direction (downhill)
• In simple terms, we assign random values for all the weights, then 
iteratively update them by following the direction of steepest descent 
(downhill).

Variants of Gradient Descent
▪Update the parameters 𝑊 based on a subset of the training examples (e.g. 
even one training example).
▪An optimization algorithm with one training example per iteration is called a 
stochastic Gradient Descent or online algorithm.
▪
Gradient = computed from a single training sample
▪
Fastest per iteration
▪
Noisiest gradient estimate
▪Typically, in deep learning we use more than one but less than all the training 
examples per iteration. In other words, we use batches of the training set 
(called mini-batches stochastic descent).
▪
Gradient = average over a small batch
▪
Good balance between speed and accuracy
▪
Most commonly used in practice
▪If not specified clearly, many people use (SGD) to refer to the mini-batches 
stochastic descent. In this course, we follow this convention if not clearly 
stated otherwise.

SGD + Momentum
Key idea: imagine we have a heavy metal ball rolling on a surface.
•
The surface represents the cost function, 
and the ball represents the weight 
updates
•
The ball starts rolling by following the 
direction of steepest descent (the 
gradient), gaining some momentum.
•
Once it has gained velocity, the direction 
in which it rolls is not only affected by the 
direction of steepest descent, but also by 
the momentum it has gained.
•
The momentum tries to keep the ball 
rolling in the direction it was moving in 
historically.

23
Momentum helps in the following ways:
Damps the oscillation in directions with high curvature
▪
When the gradient keeps changing direction, momentum acts like a damper. It averages 
out these opposing movements. The accumulated momentum counteracts sudden direction 
changes, smoothing out the zigzag path. 
▪
The heavy ball does not instantly change direction when it hits a steep wall. Its momentum 
carries it forward more steadily instead of bouncing erratically
Builds speed in directions with gentler gradients.
▪
When gradients consistently point in the same direction (even if small), momentum 
accumulates over time, building up speed. Each iteration adds a bit more velocity in that 
direction
▪
The heavy ball might start rolling slowly on a gentle slope, but it keeps accelerating as 
momentum builds up, eventually moving much faster than a single gradient step would 
allow.

AdaGrad
▪Short for Adaptive Gradient, AdaGrad is an optimization algorithm 
that it has a similar idea to momentum, but instead we keep track of 
historical squared sum of gradients. 
▪Dividing the update value by the square root of grad_squared 
improves learning by adapting the gradient updates for each weight 
based on how it was updated historically. 
grad_squared = 0
for t in range(1, num_iterations):
minibatch = get_sample_data(data, batch_size)        
dW = compute_gradients(W)
grad_squared += dW * dW
W -= learning_rate * dW / (sqrt(grad_squared) + epsilon)
Note: we add 
epsilon=1e-7 
to prevent 
division by 0

RMSProp (Leaky AdaGrad)
▪Stands for Root Mean Squared Propagation
▪ Unlike regular AdaGrad where all past gradients accumulate forever, 
RMSProp allows old gradient information to gradually "leak away" over time
▪It uses a decay rate to decay the running average of squared gradients. Can 
think of it as ‘friction’.
▪
If decay_rates= 0, Information from iteration 1 is still fully present at iteration 1000 (traditional AdaGrad)
▪
If decay_rates= 0.9, Information from many iterations ago gets multiplied by 0.9 × 0.9 × 0.9... many 
times. 
•
Eventually becomes negligibly small - it has "leaked out"

Adam Optimizer (RMSProp+Momentum)
▪The Adam optimization algorithm (derived from ‘adaptive moment 
estimation’) combines the ideas of momentum and RMSProp, and 
works extremely well in practice
▪However, what happens at the beginning, when the second moment is 
initialized to 0 and say beta2 is set to 0.999?
▪The first time we compute second_moment, it will be a value close to 0, and in the update 
rule, we also divide by the square root of this small number.
▪This means we will have initially have very large updates while the second moment is 
initially close to 0. This means very large first step.
▪This is not a major issue for first_moment because we multiply by it (it is in the numerator)
Naïve implementation, directly combine RMSProp with momentum
first_moment = 0
second_moment = 0
for t in range(1, num_iterations):
minibatch = get_sample_data(data, batch_size)        
dW = compute_gradients(W)
first_moment = beta1 * first_moment + (1 - beta1) * dW
second_moment = beta2 * second_moment + (1 - beta2) * dW * dW
W -= learning_rate * first_moment / (sqrt(second_moment) + epsilon)

Model Evaluation
▪We can train a model using any optimization algorithm to 
minimize the cost function over the training data, but how do we 
know how good the model will perform in the real world?
▪How do we measure performance in the “real world”? We 
can test the model against examples that it has not seen 
during training, and see how it performs on those.
▪It is often infeasible to acquire new data just for testing a model, 
so the standard practice is to split the dataset we already have 
into a training set and test set. 
▪In traditional ML, the training set usually accounts for 60-
80% of the dataset. However, in the era of deep learning and 
big data, datasets can consist of millions of training 
examples, and so the training set can be upwards of 90% of 
the dataset

---PAGE---
38
Model Evaluation
▪We usually measure model performance using accuracy, but we 
can also use other metrics such as precision, recall, or many 
similar metrics depending on the application. 
▪However, whichever metric we choose, we measure it on both 
the training set and the test set. We can then compare these two 
metrics to see how our model is performing:
▪If the training accuracy and test accuracy are roughly the same, 
then it means our model generalizes well
▪If there is a large gap between the training accuracy and the 
test accuracy, then our model is not generalizing well, and has 
adapted too much to the training data.

Bias and Variance
▪
Bias = How far off your model's average predictions are from the true values
▪
Caused by: Model is too simple to capture the underlying pattern
▪
Result: Underfitting - poor performance on both training and test data
▪
Example: Using a straight line to fit a sine wave (like the left plot on previous slide)
▪
Variance = How much your model predictions change if you use it on 
different datasets
▪
Caused by: Model is too complex and “memorizes” noise/outliers
▪
Result: Overfitting - great on training data, terrible on new test data
▪
Example: 12th-degree polynomial with wild oscillations (like the right plot on previous slide)
▪The bias-variance tradeoff is a fundamental concept in machine learning

Scenarios Where Regularization Is Needed
▪There are several common scenarios where regularization can be useful:
▪High Feature Count: When you have many features (variables) in your 
dataset, there’s a risk of overfitting. Regularization helps control the 
complexity of your model and prevents it from becoming too sensitive to 
the specifics of the training data.
▪Small Dataset: If your dataset is small, it’s easier for a model to overfit 
the limited data. Regularization can help balance the model’s fit to the 
training data and its ability to generalize to new, unseen examples.
▪Noisy Data: Data with a lot of noise can cause models to capture the 
noise instead of the true signal. Regularization helps by keeping the 
model’s complexity in check, preventing it from memorizing the noise.
▪High Model Complexity: Highly complex models, such as deep neural 
networks, can easily overfit the training data. Regularization techniques 
like dropout and weight decay can help these models generalize better.

Regularization Functions: L2
The main functions commonly used for regularization are L1 regularization 
and L2 regularization:
▪L2 Regularization: 𝑅𝑊= σ𝑖,𝑗𝑊𝑖𝑗
2
▪It is the sum of the squares of all the weights
▪Penalizes large weights: since the weights are squared, bigger weights are 
penalized more heavily than smaller ones. This results in the weights being more 
spread out, rather than relying on any subset of the weights.
▪Handles multi-collinearity: if any input features of x are highly correlated, the 
weights assigned to those features would be large, which results in unstable 
outputs (high variance). L2 Regularization shrinks the weights of any correlated 
features equally, leading to a more stable model.
▪Overall more stability: by keeping all the weights small and similar in scale, L2 
helps a model behave more predictably on new data, since any noisy training 
examples cannot force the weights to change in magnitude by a lot.

Regularization Functions: L1
▪L1 Regularization: 𝑅𝑊= σ𝑖,𝑗|𝑊𝑖𝑗|
▪i.e. the sum of the absolute values of all the weights
▪Enables feature selection: L1 regularization shrinks some weights to 0, 
which means that only the most important features are considered in the 
final output of the model. This is a way to perform feature selection.
▪Maintains interpretability: due to the fact that some weights may be 0, this 
reduces the number of features used in the output, which makes it easier to 
interpret. Furthermore, since only the important features are used, we can 
clearly see which features actually affect our output.

𝑥𝑇𝑤1 = 1 × 1 + 0 × 1 + 0 × 1 + 0 × 1 = 1

𝑅𝑤1 = 1 2 + 0 2 + 0 2 + 0 2 = 1
𝑠𝑜 𝑔(𝑤) = 𝑑𝑎𝑡𝑎 𝑙𝑜𝑠𝑠+ 1

𝒘𝟐 is better

Both weight sets make the same prediction
•

L2 regularization prefers spreading weight across all features rather than

Backpropagation (Reverse-Mode AD)
в–ӘBackpropagation is also known as reverse-mode automatic 
differentiation (reverse-mode AD): compute gradients of a scalar 
loss рқҗҝ w.r.t. all weights/biases efficiently.
в–ӘKey idea: each layerвҖҷs backward step is a vectorвҖ“Jacobian 
product (VJP); overall cost is a small constant times a forward 
pass (e.g. If a forward pass takes ~1 ms, then forward plus 
backward might take ~2вҖ“3 ms.).
в–ӘTwo-phase algorithm:
в–ӘForward pass: compute layer activations and cache them for 
derivatives calculations.
в–ӘBackward pass: propagate error signal backward using 
vectorвҖ“Jacobian products.

вҖў Backpropagation is an algorithm used to compute gradients; it

вҖў Backpropagation is used to compute different types of gradients

вҖў We need another algorithm (e.g. optimization algorithm like 
SGD) to perform learning using these gradients.
вҖў SGD will use the gradients to update both the weights and the 
biases 
вҖў Backpropagation is not specific to neural networks; it can be

Forward vs. Backward Pass
в–ӘThere is a difference between the forward and backward passes:
в–ӘIn the forward pass we use squashing non-linear activation functions (like 
sigmoid, tanh, ReLU) that bound the outputs 
вҖў e.g, sigmoid maps any input to (0, 1)
вҖў Prevent activations from exploding as signals flow forward through layers
в–ӘDuring backpropagation, gradients flow backward through the network
в–ӘThe backward pass, is completely linear. Even though the forward pass 
is non-linear, backpropagation is a linear operation.
вҖў  If you double the error derivatives at the final layer, all the error derivatives 
will double. 
в–ӘThe forward pass computes activations рқ‘Һ= рқ‘“(рқ‘§). During backprop, we 
calculate рқӣҝрқ‘“= рқӣҝрқ‘“

The Problem of Vanishing/Exploding Gradients
в–ӘThis repeated multiplication can cause gradients to either vanish or explode
в–ӘVanish (вҶ’ 0) if local gradients < 1
вҖў When local gradients are small, this is called saturation
в–ӘExplode (вҶ’ вҲһ) if local gradients > 1
в–ӘTypical shallow feed-forward neural nets can cope with these exponential 
effects because they only have a few hidden layers.
в–ӘPractical symptoms:
в–ӘVanishing: early layers learn very slowly, training plateaus
в–ӘExploding: loss becomes NaN/Inf, unstable updates

в–ӘCauses for Vanishing Gradient:
в–ӘThe chain rule repeated multiplication 
    can cause gradients to vanish
вҖў Vanish (вҶ’ 0) if local gradients < 1
в–ӘActivation functions:
вҖў Sigmoid: рқңҺвҖІ рқ‘ҘвҲҲ0, 0.25
вҖў Tanh: рқ‘Ўрқ‘Һрқ‘ӣв„ҺвҖІ рқ‘ҘвҲҲ0, 1
вҖў Derivatives < 1 multiplied through the layers
вҖў Depth makes it worse: repeated multiplication by numbers < 1
в–ӘPoor weight initialization: Small random weights can lead to a combined 
effect of gradient vanishing
в–ӘCauses for Exploding Gradient:
в–ӘThe chain rule repeated multiplication can cause gradients to explode
вҖў Explode (вҶ’ вҲһ) if local gradients > 1
в–ӘPoor weight initialization: Large random weights combined with no proper 
normalization can lead to gradient exploding

# transforms.Compose: Chains multiple transformations together into a single pipeline 
# The transformations are applied sequentially to each image 
transform = transforms.Compose([ 
   # transforms.ToTensor(): Converts PIL Image or numpy.ndarray (H x W x C) in range [0, 255] 
   # to a torch.FloatTensor of shape (C x H x W) in range [0.0, 1.0]  
   transforms.ToTensor(), 
   # transforms.Normalize(): Normalizes tensor with mean and standard deviation 
 
   # Formula: output = (input - mean) / std 
   # Here: (0.5,) is the mean and (0.5,) is the std for single channel (grayscale) 
      # This transforms values from [0, 1] to [-1, 1] range

# NEURAL NETWORK ARCHITECTURE DEFINITION # 
#################################################################
# nn.Module: Base class for all neural network modules in PyTorch 
# Your models should subclass this class 
class Net(nn.Module): 
  def __init__(self): 
    # super().__init__(): Calls the constructor of the parent class (nn.Module) 
    # This is required to properly initialize the module 
    super(Net, self).__init__()
    # nn.Linear: Applies a linear transformation: y = xW^T + b 
    # Parameters: 
    # - in_features: Size of input sample (28*28 = 784 for flattened MNIST image) 
    # - out_features: Size of output sample (512 neurons in this layer) 
    # This creates a weight matrix of shape (512, 784) and bias vector of shape (512,) 
    self.fc1 = nn.Linear(28*28, 512) # First fully connected layer 
    self.fc2 = nn.Linear(512, 256) # Second fully connected layer 
    self.fc3 = nn.Linear(256, 10) # Output layer (10 classes for digits 0-9)
  # Defines the forward pass of the network. This method is called when you pass data through the model 
  # Args: x: Input tensor of shape (batch_size, 1, 28, 28) 
  #Returns: Output tensor of shape (batch_size, 10) with raw scores for each class
  def forward(self, x): 
    # x.view(): Reshapes the tensor without changing its data 
    # -1 means "infer this dimension from other dimensions" 
    # Converts (batch_size, 1, 28, 28) to (batch_size, 784) 
    x = x.view(-1, 28*28) 
    # Pass through first fully connected layer 
    x = self.fc1(x)
    # torch.relu(): Applies Rectified Linear Unit activation function 
    # ReLU(x) = max(0, x) - zeros out negative values 
    # This introduces non-linearity, allowing the network to learn complex patterns 
    x = torch.relu(x) 
    # Pass through second fully connected layer 
    x = self.fc2(x) 
    # Apply ReLU activation 
    x = torch.relu(x) 
    # Pass through output layer 
    # Note: No activation here because CrossEntropyLoss applies softmax internally 
    x = self.fc3(x) 
 
    return x

# TRAINING FUNCTION # 
#################################################################
#Train the neural network 
#Args: model: NN model to train criterion: Loss function (e.g., CrossEntropyLoss) 
#optimizer: Optimization algorithm (e.g., SGD, Adam) 
# epochs: Number of complete passes through the training dataset
def train_model(model, criterion, optimizer, epochs=3): 
  # Loop over the dataset multiple times 
  for epoch in range(epochs): 
    running_loss = 0.0 
    # Accumulator for loss values # enumerate(): Returns index and data from the iterable 
    # trainloader yields batches of (images, labels) 
    # The second parameter (0) is the starting index 
    for i, data in enumerate(trainloader, 0): 
      # Unpack the batch 
      inputs, labels = data # inputs: (64, 1, 28, 28), labels: (64,) 
      # optimizer.zero_grad(): Clears gradients of all optimized tensors 
      # PyTorch accumulates gradients by default, so we need to clear them 
      # before computing gradients for the current batch 
      optimizer.zero_grad() 
      # Forward pass: compute predicted outputs by passing inputs to the model 
      outputs = model(inputs) # Shape: (64, 10) 
      # criterion(): Computes the loss between predictions and true labels 
      # CrossEntropyLoss combines nn.LogSoftmax() and nn.NLLLoss() 
      loss = criterion(outputs, labels) 
      # loss.backward(): Computes gradients of loss with respect to model parameters 
      # Uses automatic differentiation (backpropagation) 
      loss.backward() 
      # optimizer.step(): Updates model parameters using computed gradients 
      # Each optimizer (SGD, Adam, etc.) has its own update rule 
      optimizer.step() 
      # loss.item(): Extracts the loss value as a Python float 
      # This is used for monitoring/logging (doesn't require gradients) 
      running_loss += loss.item() 
      # Print statistics every 100 mini-batches 
      if i % 100 == 99: 
        # Print epoch number, batch number, and average loss 
        print('[%d, %5d] loss: %.3f' % (epoch + 1, i + 1, running_loss / 100)) 
      running_loss = 0.0 # Reset running loss

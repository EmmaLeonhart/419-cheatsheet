CNN Structure
▪CNN structure is inspired from the biological structure of our 
visual cortex. 
▪A study showed that cells in the visual cortex are found to 
activate based on sub-regions of the visual field. Same idea is 
used in CNN where neurons are connected to sub-regions, and 
they are not responsive to areas outside these sub-regions.
Source: mathworks

Weight Sharing
What is Weight Sharing?
▪In CNNs, the same filter (kernel) weights are reused across all spatial positions of the input.
 Key Differences:
▪Fully Connected: Every input-output connection has a unique weight → many parameters
▪CNN (Convolutional): Same filter slides across input, reusing weights → much fewer parameters
 A larger Example: 28×28 input → 100 neurons
▪Fully connected: 78,400 parameters
▪CNN with 32 3×3 filters: 288 parameters (270× reduction!)

Benefits
▪ Same Filter, Multiple Positions:
▪A 3×3 filter with 9 weights is applied to every position in the input
▪For a 28×28 input: filter is applied 26×26 = 676 times, but only 9 weights 
stored
▪Translation Invariance:
▪Same filter detects the same pattern (edge, texture, shape) anywhere in 
the image
▪A circle detector at top-left will also detect circles at bottom-right
▪Training with Weight Sharing:
▪During backpropagation, gradients from ALL positions accumulate
▪Single filter update: w = - 𝛼 Σ(∇w from all 676 positions) 
▪More on this later
▪Filter learns from every location it was applied → better generalization
▪ Massive parameter reduction + translation invariance + robust feature 
learning

Questions for Discussion
▪How are the filters related to the weights in a typical MLP 
network?
▪A convolution filter is a set of learnable weights. The difference is weight sharing: the 
same small kernel is applied across all spatial locations instead of having separate weights 
per pixel.
▪How are the filters related to features?
▪A filter becomes a feature detector after training. Convolving it over the image produces a 
feature map (activation map) showing where that feature appears and how strongly
▪Why are the filters in the previous image colored?
▪Because for an RGB input, each filter has depth 3 (one kernel per input channel)
▪What happens if we stack two convolution layers back-to-back?
▪Stacking conv layers increases the effective receptive field and enables hierarchical 
composition (simple features → more complex ones). Two small kernels (e.g., 3 × 3+ 3
× 3) can behave like a larger one (≈5 × 5)

Common Types of Pooling
▪Max pooling: Takes the maximum value within a local region, 
which is the most commonly used pooling method.
▪Average pooling: Calculates the average value within a local 
region.

---PAGE---
43
Why We Need Pooling
▪Dimensionality reduction: By aggregating information within a 
local area, pooling layers downsample the input, significantly 
reducing the number of parameters in the network.
▪Computational efficiency: Reducing the size of feature maps 
decreases the computational cost of subsequent layers.
▪Translation invariance: Pooling makes the model less sensitive 
to small shifts in the input image, as the most important feature 
within a local region will still be captured.
▪Feature extraction: By selecting the maximum or average value 
within a region, pooling helps extract the most salient features 
from the convolutional layers.

How stride and padding affect Δ𝑊 during backprop
▪Key idea: each filter’s weight update ΔW is built by summing contributions from many 
input “patches” (the parts of 𝑋 the filter slides over).
▪Stride changes which patches are used
▪Bigger stride = filter “jumps” more.
▪That means fewer output positions, so you sum over fewer patches.
▪Result: Δ𝑊 changes because the filter sees a different set of patches.
▪Padding changes the patches near the border
▪More padding adds zeros around the input.
▪Border output positions now use patches that include some zeros (different patch 
content).
▪It can also change output size (more positions to sum over).
▪Result: Δ𝑊 changes because the patches (and/or number of patches) change.
▪Conclusion
▪Changing stride or padding changes the forward sliding pattern, 
▪So, it changes the sum used to compute 𝑑𝐿/𝑑𝑊,
▪Therefore, it changes Δ𝑊 for all filters (except rare “everything is zero” cases).

Deep CNNs Have Gradient Issues
▪CNNs still train with backprop + chain rule
▪As we stack more conv blocks:
▪gradients can vanish (slow learning in early layers)
▪gradients can explode (instability / divergence)
▪If we have vanishing or exploding gradients the network cannot 
learn and become instable
▪This motivates common CNN “training stabilizers”:
▪normalization using BatchNorm (see next slides)
▪residual connections as in ResNet architecture (next lecture)
▪careful initialization (next next lecture)

Batch Normalization
▪A method used to make training of artificial neural networks 
faster and more stable through normalization of the layers inputs 
by re-centering and re-scaling.
▪The idea behind batch normalization is to try to tackle a problem 
called the internal covariate shift problem
▪In short, batch normalization aims to normalize the input by 
subtracting and dividing by the mini-batch statistics.

Why BatchNorm often makes deep CNN training easier
▪BatchNorm doesn’t “magically remove” vanishing gradients in every case, but 
it strongly reduces the main causes of vanishing/exploding in deep nets by 
controlling activation scale
▪BatchNorm normalizes pre-activations within a mini-batch:
▪BN makes each layer’s inputs roughly zero mean, unit variance, then lets 
the network learn the best scale/shift via 𝛾, 𝛽.

---PAGE---

This helps with vanishing gradients by:
▪Keeps activations in a “high-derivative”. For example, the Sigmoid saturates for 
large ∣𝑧∣, making 𝜎′ 𝑧≈0 ;keeping pre-activations near 0 keeps 𝜎′ 𝑧 near its 
maximum (~0.25), improving gradient flow through depth.
• Even with ReLU, controlling scale matters because too-small variance yields tiny 
signals/gradients, and too-large variance can destabilize training.
▪Stabilizes gradient magnitudes through scale control: BN introduces a 
normalization factor 1/ 𝜎𝐵
2 + 𝜖 .Very roughly (intuition, not exact equality), 
gradients backpropagated through BN get rescaled in a way that prevents them 
from blowing up or shrinking purely due to drifting activation variance
▪Improves conditioning of the optimization problem: BN often makes the loss 
landscape “easier” for SGD-like optimizers. This is part of why you can often use 
higher learning rates with BN.

▪How BN reduces dependence on weight initialization
▪Good initialization (Xavier/He) tries to keep activation/gradient variance 
roughly stable at the start. But without BN, during training the 
distributions can drift, and the network can become very sensitive to 
initial scale.
▪BN reduces this sensitivity because:
▪Even if your weights start with a “bad” scale, BN renormalizes the 
resulting pre-activations to a standard scale each forward pass.
▪BN reduces sensitivity to weight initialization because weight scaling 
largely cancels in normalization (then 𝛾, 𝛽 learn the right scale/shift).
▪That means the network is less sensitive to the exact initial weight 
scale.
▪Authors have run extensive testing as seen on next slides to prove these 
claims

Overall
▪Pros:
▪BN reduces Covariate Shift. That is the change in distribution of 
activation of a component. By using BN, each neuron's 
activation becomes (more or less) a Gaussian distribution, i.e. 
its usually not active, sometimes a bit active, rare very active.
▪Makes deep networks much easier to train (see next slides)
▪Allows higher learning rates, faster convergence 
▪Networks become more robust to initialization
▪Acts as regularization during training
▪Zero overhead at test-time: can be fused with conv layers
▪Cons:
▪Not well-understood theoretically
▪Behaves differently during training and testing (can cause 
issues if not careful)

Layer Normalization
▪Another type of normalization is called Layer Normalization (LayerNorm). 
▪BatchNorm is often the default choice in CNNs because it normalizes per 
channel using mini-batch (and spatial) statistics, and it often improves 
training speed and final accuracy when the batch size is reasonably large.
▪We will discuss LayerNorm in more detail when we cover 
Transformers/attention, because they use it as the standard normalization 
in Transformer blocks.
▪LayerNorm does not rely on batch statistics, which is helpful for variable-
length sequences and small batches
▪Because LayerNorm computes its mean/variance within each individual 
example (batch=1), it still works consistently when:
• sequences have variable length, and
• batch size is small (batch statistics become noisy/unreliable).

# Define a simple neural network
class Net(nn.Module):
  def __init__(self, use_batch_norm=False):
    super(Net, self).__init__()
    self.use_batch_norm = use_batch_norm
    self.fc1 = nn.Linear(28*28, 512)
    self.fc2 = nn.Linear(512, 256)
    self.fc3 = nn.Linear(256, 10)
    # Batch Normalization layers
    if use_batch_norm:
      self.bn1 = nn.BatchNorm1d(512)
      self.bn2 = nn.BatchNorm1d(256)

   x = x.view(-1, 28*28)
   x=self.fc1(x)
   # Apply batch normalization if specified
   if self.use_batch_norm:
      x = self.bn1(x)
   x=torch.relu(x)
   x=self.fc2(x)  
   
   # Apply batch normalization if specified
   if self.use_batch_norm:
      x = self.bn2(x)
   x=torch.relu(x)
   
   x = self.fc3(x)

def train_model(model, criterion, optimizer, epochs=3):
  for epoch in range(epochs):
    running_loss = 0.0
    for i, data in enumerate(trainloader, 0):
      inputs, labels = data
      optimizer.zero_grad()
      outputs = model(inputs)
      loss = criterion(outputs, labels)
      loss.backward()
      optimizer.step()
      running_loss += loss.item()
      if i % 100 == 99:
        print('[%d, %5d] loss: %.3f' %
           (epoch + 1, i + 1, running_loss / 100))
        running_loss = 0.0

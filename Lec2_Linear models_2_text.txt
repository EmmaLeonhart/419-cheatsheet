Points on the negative side of all classifiers
•
In this case we cannot argue that one classifier is more 
'confident' in the class identity of such points. However, we can 
ask which classifier is the least 'unsure' about the class identity 
of such points
•
Since all classifiers are saying "this point is NOT in my class," 
we apply a different logic:
•
Pick the classifier with the LEAST NEGATIVE value 
(i.e., the largest among negative values)

•

𝑥𝑇𝑤𝑗 = - 5.0: "I'm very far into the negative side, definitely NOT my class"

The classifier closest to zero (least negative) is:

Loss: Softmax Regression
• We are still dealing with the case of more than two classes, i.e. y(i) ∈ {0,…,C-1}  where C is 
the number of classes. Therefore, Softmax regression (multi-class classification; C > 2) is 
a generalization of logistic regression (binary classification; C = 2).
• Now our training set of P labeled examples looks like { 𝒙𝟏, 𝒚𝟏, 𝒙𝟐, 𝒚𝟐, … , 𝒙𝑷, 𝒚𝑷} 
• Each input 𝒙𝒊 ∈ RN i.e. each 𝒙𝒊 is an N-dimensional vector representing N features;
• Each output 𝒚𝒊 ∈ {0,…,C-1} i.e. each 𝒚𝒊 is a discrete scalar value between 0 to C-1. 
For example, in MNIST digit recognition, we have C = 10 different classes.
•
We can use the LogSumExp approximation, which is a smooth approximation of the max 
function:
max { 𝑠0, 𝑠1, … , 𝑠𝐶−1 } ≈ LogSumExp(𝑠0, 𝑠1, … , 𝑠𝐶−1 ) ≈ log( 𝑒𝑠0 + 𝑒𝑠1 + ⋯ + 𝑒𝑠𝐶−1)

The multi-class perceptron cost uses the max function, which is not differentiable at the 
point where two values are equal. This makes optimization difficult.
•
Also, the multi-class perceptron cost function is not smooth
•
We need to replace it with a smoother function.

Why It is Called Softmax
•
We want to estimate the probability of the class label taking on each of the 
C different possible values. Thus, the final output of the classifier should be a 
C-dimensional vector, whose elements sum to 1. This will give us the 
estimated probabilities for each class. 
•
To output a probability distribution, we can use the Softmax function, which 
takes as input some C-dimensional vector, say z:
•
where each 𝑧𝑖 represents one element in the input vector z. 
•
In our case, each 𝑧𝑖= 𝑥𝑇𝑤𝑖. 
•
In the denominator, we sum over all elements of the input vector z. Dividing by 
this sum normalizes values in the output vector to be between 0 and 1, which 
makes it a probability distribution!

Code: Activation Functions in PyTorch
PYTHON / PYTORCH
import torch
import torch.nn as nn
# Defining activation functions
relu = nn.ReLU()
leaky_relu = nn.LeakyReLU(negative_slope=0.01)
gelu = nn.GELU()
# Using in a model
class SimpleNet(nn.Module):
def __init__(self, input_dim, hidden_dim, num_classes):
super().__init__()
self.fc1 = nn.Linear(input_dim, hidden_dim)
self.act = nn.GELU()           # Recommended default
self.fc2 = nn.Linear(hidden_dim, num_classes)
def forward(self, x):
x = self.act(self.fc1(x))      # Apply activation after 
linear
return self.fc2(x)
Recommendations
Default choice: ReLU
Fast, simple, works well for most CNNs. 
Set lr carefully.
For transformers: GELU
Smooth gradient near zero, used in 
BERT, GPT, ViT.
Dying neurons? Leaky ReLU
If many dead units (output always 0), 
switch to LeakyReLU or PReLU.
Avoid: Sigmoid / tanh
Vanishing gradients in deep nets. Only 
use in output layers (e.g. sigmoid for 
binary classification).

Code: Weight Initialization in PyTorch
PYTHON / PYTORCH
import torch.nn as nn
import torch.nn.init as init
class ConvNet(nn.Module):
def __init__(self):
super().__init__()
self.conv1 = nn.Conv2d(3, 64, 3, padding=1)
self.conv2 = nn.Conv2d(64, 128, 3, padding=1)
self.fc = nn.Linear(128 * 8 * 8, 10)
self._init_weights()
def _init_weights(self):
for m in self.modules():
if isinstance(m, nn.Conv2d):
# Kaiming init for ReLU networks
init.kaiming_normal_(m.weight, mode='fan_out',
nonlinearity='relu')
elif isinstance(m, nn.Linear):
# Xavier init for linear layers
init.xavier_normal_(m.weight)
init.zeros_(m.bias)
When to Use What
ReLU / LeakyReLU layers:
Kaiming (He) initialization
std = sqrt(2 / fan_in)
Sigmoid / tanh layers:
Xavier (Glorot) initialization
std = sqrt(1 / fan_in)
Transformers / LayerNorm:
Usually handled by the framework (e.g. 
nn.Transformer). Use default init.
Never use constant init (all same value) 
— breaks symmetry requirement.

Code: Dropout & Regularization
PYTHON / PYTORCH
import torch.nn as nn
class RegularizedNet(nn.Module):
def __init__(self):
super().__init__()
self.features = nn.Sequential(
nn.Linear(784, 512),
nn.ReLU(),
nn.Dropout(p=0.5),       # Drop 50% during training
nn.Linear(512, 256),
nn.ReLU(),
nn.Dropout(p=0.3),       # Less dropout in later 
layers
nn.Linear(256, 10)
)
def forward(self, x):
return self.features(x)
# IMPORTANT: toggle train/eval mode
model.train()   # Enables dropout (training)
model.eval()    # Disables dropout (inference)
# Weight decay (L2 regularization) in optimizer
optimizer = torch.optim.Adam(model.parameters(),
lr=1e-3, weight_decay=1e-4)
Regularization Checklist
Dropout rates:
FC layers: p = 0.5 (typical)
Conv layers: p = 0.1–0.3
After attention: p = 0.1
Weight decay:
Start with 1e-4, try 1e-5 or 0
Common mistake:
Forgetting model.eval() at test time! This 
keeps dropout on and produces noisy 
predictions.
Order of operations:
Linear → BatchNorm → Activation → 
Dropout

Code: Data Augmentation Pipeline

from torchvision import transforms
# Training transforms (apply augmentation)
train_transform = transforms.Compose([
transforms.RandomResizedCrop(224, scale=(0.8, 1.0)),
transforms.RandomHorizontalFlip(),
transforms.ColorJitter(brightness=0.2, contrast=0.2,
saturation=0.2, hue=0.1),
transforms.RandomErasing(p=0.25),  # Cutout-like
transforms.ToTensor(),
transforms.Normalize(mean=[0.485, 0.456, 0.406],
std=[0.229, 0.224, 0.225]),
])
# Validation transforms (NO augmentation)
val_transform = transforms.Compose([
transforms.Resize(256),
transforms.CenterCrop(224),
transforms.ToTensor(),
transforms.Normalize(mean=[0.485, 0.456, 0.406],
std=[0.229, 0.224, 0.225]),
])
Key Points
Always augment training only
Validation/test data should use 
deterministic transforms.
Start simple:
Flip + Crop + Normalize is a strong 
baseline.
Add progressively:
ColorJitter, RandomErasing, or 
AutoAugment if underfitting is not a 
problem.
Domain matters:
Medical images: don't flip vertically if 
orientation matters. Satellite: rotation is 
fine. Text: no horizontal flip.

Optimization toolkit: mini-batch SGD
We optimize parameters θ to minimize loss L(θ) using gradients 
∇θL
Mini-batch SGD baseline: compute gradient on a batch B and 
update
θ ← θ − η · (1/|B|) Σ_{i∈B} ∇θ L_i
Why mini-batches? Efficient (vectorized), less noisy than batch 
size 1, cheaper than full batch
Key knobs: learning rate η, batch size |B|, weight decay / 
regularization

---PAGE---
66
Momentum & Adam (practical defaults)
Momentum smooths noisy gradients and accelerates along 
consistent directions
v ← μv − η∇θL   ;   θ ← θ + v   (μ≈0.9)
Adam ≈ Momentum + per-parameter step sizes (RMS of 
gradients) (+ bias correction)
Good starting point: Adam (η≈1e−3) or SGD+Momentum (η 
tuned, μ≈0.9)
If unstable: lower η first; if too slow: raise η or add momentum

---PAGE---
67
Batch size: speed vs generalization
Small batches: noisier updates → can help generalization, but 
slower / less stable
Large batches: efficient hardware utilization, smoother gradients, 
may generalize worse
Rule of thumb: start with 64–256 (fit to memory), then tune
Batch size interacts with learning rate (often increase η when 
increasing batch size)

---PAGE---
68
Learning-rate schedules (why decay helps)
Large η helps make fast progress early; smaller η helps “settle” 
near a good solution
Common schedules:
Step decay (drop η by 10× at milestones)
Cosine decay (popular default), often with short warm-up
Exponential / 1/t
Recipe: get training working with constant η → then add decay 
for best accuracy

---PAGE---
69
Training curves: diagnose → fix
Plot training & validation loss/accuracy; keep a fixed eval subset 
to reduce noise
Loss explodes / oscillates → decrease η; check init; try Adam / 
momentum
Loss decreases very slowly → increase η; add momentum; 
check data preprocessing
Train improves but val worsens → overfitting: regularize / 
augment / early stop
Both are bad → underfitting: bigger model, train longer, reduce 
regularization

import matplotlib.pyplot as plt
def plot_training_curves(train_losses, val_losses,
train_accs, val_accs):
fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(12, 4))
ax1.plot(train_losses, label='Train Loss')
ax1.plot(val_losses, label='Val Loss')
ax1.set_xlabel('Epoch'); ax1.set_ylabel('Loss')
ax1.legend(); ax1.set_title('Loss Curves')
ax2.plot(train_accs, label='Train Acc')
ax2.plot(val_accs, label='Val Acc')
ax2.set_xlabel('Epoch'); ax2.set_ylabel('Accuracy')
ax2.legend(); ax2.set_title('Accuracy Curves')
plt.tight_layout(); plt.savefig('curves.png')
plt.show()
What to Look For
✔Loss decreasing smoothly
Training is healthy.
⚠Train loss ↓ but val loss ↑
Overfitting! Add regularization, dropout, data 
augmentation, or reduce model size.
⚠Both losses plateau early
Underfitting. Increase model capacity, train longer, or 
raise learning rate.
❌Loss explodes / NaN
LR too high or bad init. Reduce LR by 10x, check 
initialization.
❌Loss stays flat from start
LR too low or bug. Increase LR, verify data loading 
and labels.

Putting It All Together: Training Loop
COMPLETE EXAMPLE
import torch
import torch.nn as nn
import torch.optim as optim
# --- Setup ---
model = ConvNet()                                # Use proper weight init (Kaiming)
optimizer = optim.Adam(model.parameters(), lr=1e-3, weight_decay=1e-4)  # L2 reg
scheduler = optim.lr_scheduler.CosineAnnealingLR(optimizer, T_max=num_epochs)
criterion = nn.CrossEntropyLoss()
# --- Training loop ---
best_val_acc = 0
for epoch in range(num_epochs):
model.train()                                # Enable dropout / batchnorm train mode
for images, labels in train_loader:
outputs = model(images)
loss = criterion(outputs, labels)
optimizer.zero_grad()
loss.backward()
optimizer.step()
# --- Validation ---
model.eval()                                 # Disable dropout for evaluation
with torch.no_grad():
val_loss, correct, total = 0, 0, 0
for images, labels in val_loader:
outputs = model(images)
val_loss += criterion(outputs, labels).item()
correct += (outputs.argmax(1) == labels).sum().item()
total += labels.size(0)
val_acc = correct / total
scheduler.step()
print(f"Epoch {epoch}: train_loss={loss:.4f}, val_acc={val_acc:.4f}, lr={scheduler.get_last_lr()[0]:.6f}")
if val_acc > best_val_acc:                   # Save best model
best_val_acc = val_acc
torch.save(model.state_dict(), 'best_model.pth')
